Rules

What this system does, and what it refuses.

The system keeps content in charge.

  • Warm fields — use tinted neutrals and soft surface separation, not stark white.
  • Typography-led hierarchy — spacing, weight, and font role carry structure before color does.
  • Accent scarcity — deep moss green appears at links, focus, and active state only. Never as a section wash.
  • Mono precision — IBM Plex Mono for metadata, tags, and labels. Never for headings.
  • Flat at rest — shadow as response, not as permanent decoration.
  • Scale lock — five type sizes, no custom in-between steps.
  • Focus always visible — 2px accent outline with offset on every interactive element.

The system refuses these.

  • Glossy SaaS hero metrics or generic feature grids.
  • Neon-on-black devtool theatrics.
  • Decorative serif flourishes or editorial costume.
  • Glassmorphism, gradient text, or retro terminal framing.
  • Backdrop-filter blur as decoration (functional nav only).
  • Cards as the only layout pattern — vary structure per section.
